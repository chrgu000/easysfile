安装
composer.phar   linux/win通用  2进制归档文件

mac/linux 全局安装
sudo mv composer.phar /usr/local/bin/composer

win 全局安装
将composer.phar 拷贝到php.exe同级目录
新建composer.bat,并拷贝代码至文件中
@php "%~dp0composer.phar" %*

中国镜像
全局配置中国镜像
命令行输入
composer config -g repo.packagist composer https://packagist.phpcomposer.com

使用

//不常用 composer create-project --prefer-dist topthink/think think_composer(直接下载框架项目，其中dist为composer的缓存)

composer init 在当前项目目录下，配置信息。（初始化，生成json文件）

composer search  XXX（monolog）

composer show --all monolog/monolog (查看这个包的信息)

//在json文件中，require写好包名后，执行安装
composer install   

//我认为常用的安装方法
composer require symfony/httpfoundation  直接通过require来安装
composer require yiisoft/yii2-gii 2.04

composer update  在json中删除require里面的信息（monolog/monolog），执行它后进行删除

install与update
//我想得，install是以composer.lock 为基础，updata 更新composer.lock


//和自己的类相结合

先在自己的类目录里定义好命名空间
namespace easys\web;

然后往composer.json
写入
"autoload": {
		"psr-4": {
			"easys\\web\\": "src/"
		}
	}

最后在主体文件中做导入类操作
use easys\web\response;

echo response::json(20002,'sadsad');

echo easys\web\pdoobj::one();

//开启优化，这个命令的本质是将 PSR-4/PSR-0 的规则转化为了 classmap 的规则，因为 classmap 中包含了所有类名与类文件路径的对应关系，所以加载器不再需要到文件系统中查找文件了。可以从 classmap 中直接找到类文件的路径。
composer dump-autoload -o
